## 今回のご報告の趣旨

Milestone 1・Milestone 2 に引き続き、
会員さまがご自身の手で「支援の確認」「変更」「停止」「追加の寄付」までを
完結できるマイページ体験と、それを裏側で支える Stripe 決済・自動メールの仕組みを
今回の Milestone 3 で作り込みました。

これにより、貴社側で一件ずつ対応していただいていた
「支援の追加・変更・停止のご連絡」「入金確認」「お礼のご連絡」が、
会員さまの操作だけで自動的に完結する状態となっております。

## 動作確認について

本番相当環境（Next.js + Supabase + Stripe）におきまして、
Milestone 2 時点で投入した実データ（会員約 700 件、過去の支援／決済履歴）に対し、

- 支援の新規追加／口数変更／馬の変更／停止
- 単発寄付（Stripe Checkout）の実行と Webhook 受信
- 決済成功・失敗シナリオにおけるマイページ表示の切替
- お礼メール・支援変更メール・決済失敗メールの自動送信

を、すべて弊社のテスト環境にて実施・確認いたしました。

## 決済・メール送信の本番接続について（重要なお願い）

今回の Milestone 3 では、下記の 2 点を
**弊社の検証用アカウントに接続した状態**で実装・動作確認を行っております。

- Stripe（決済）
- メール送信サービス

いずれも弊社側で検証用のテストアカウントを作成し、
「決済の成功・失敗」「Webhook の受信」「お礼メールの自動配信」が
問題なく動作することを確認済みでございます。

つきましては、本番環境への切り替えにあたりまして、
下記のご対応をお願いできますと幸いです。

1. 貴社名義の **Stripe アカウント**のログイン情報のご共有
   （または招待用メールアドレスのご共有）
2. 自動お礼メールの送信元として使用する **貴社メールアカウント**のログイン情報のご共有

上記を頂戴次第、弊社側で接続切り替え作業を行い、
実際の貴社アカウントから決済・メール送信が行われる状態に
速やかに移行いたします。

お預かりいたしましたログイン情報につきましては、
**プロジェクト完了時にすべて貴社へ返却**し、
弊社側の控えも責任を持って破棄いたしますので、
安心してご共有くださいませ。
