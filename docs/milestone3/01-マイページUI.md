# 1. マイページ UI

## 概要

会員さまがログイン後、最初に到達されるマイページについて、
「今の契約状況」「現在の支援内容」「お支払い状況」「次回決済日」「月額合計」を
1 画面で把握できるダッシュボード型 UI として再構成いたしました。

今回の改修では、情報量を単純に増やすのではなく、
会員さまが迷わず次の行動を取れることを重視しております。
そのため、画面上部では状態確認に必要な情報を優先表示し、
画面中段では支援変更・支援停止・単発寄付・予約・登録情報変更などの
主要操作へ、最短で移動できる構成としております。

特に、Milestone 3 で対象となった「支援状況表示」「決済状態表示」「次回決済日表示」は、
すべてこのマイページ内で一貫した文言・色・配置で確認できるよう統一しております。

## 実装した主な機能

- 画面上部のサマリー領域に「会員種別 / 支援中頭数 / 月額合計 / 予約件数」を常時表示
- その下に「現在の会員種別」「お支払い状況」「次回決済日」「月額支援合計」を 2 × 2 で整理
- 決済失敗時には赤系、停止予定時には琥珀系、正常時には緑系の補助表示を出し、状態を直感的に把握可能
- 「支援中の馬」セクションから、各支援に対して「変更する」「停止する」を直接操作可能
- 「単発寄付」「見学会・個別見学予約」「情報を変更」「お支払い情報を変更」「履歴を見る」などの導線を整理
- 1 画面 1 目的の考え方で、スマートフォンでも無理なく読める縦積み構成へ対応

## 画面で確認できるポイント

マイページにログインすると、会員さまはまず自分のお名前と、
現在の支援状況の要約を確認できます。
この時点で「何頭を支援しているか」「月額はいくらか」「予約は何件あるか」が
一覧的に把握できるため、複数画面を行き来せずに現状を確認できます。

また、ページ中央では「お支払い状況」「次回決済日」が独立した項目として表示されるため、
会員さまにとって重要な決済情報が埋もれない構成となっております。
さらに、支援中の馬の一覧からは、支援内容の変更や停止の導線がそのまま接続されており、
閲覧と操作が分断されない UI にしております。

## 実装コード（抜粋）

マイページトップでは、会員基本情報・顧客サマリ・契約・支援一覧・直近決済履歴を
並列で取得し、表示の即時性を確保しております。

```tsx
// src/app/mypage/page.tsx
const [customer, summary, contract, supports, recentPayments] = await Promise.all([
  loadCustomer(customerId),
  loadCustomerSummary(customerId),
  loadActiveContract(customerId),
  loadActiveSupports(customerId),
  loadPayments(customerId, 3),
]);
```

画面最上部のサマリー領域では、会員さまにとって最も重要な 4 指標を
ひと目で分かる形で表示しております。

```tsx
<section className="card bg-gradient-to-br from-brand to-brand-dark text-white">
  <p className="text-sm opacity-90">こんにちは</p>
  <h1 className="text-2xl font-bold">{customer?.full_name ?? "会員"}様</h1>
  <div className="mt-3 grid grid-cols-2 sm:grid-cols-4 gap-3">
    <div>
      <p className="text-xs opacity-80">会員種別</p>
      <p className="text-base font-bold mt-1">{planBadgeText}</p>
    </div>
    <div>
      <p className="text-xs opacity-80">支援中</p>
      <p className="text-base font-bold mt-1">{supports.length}頭</p>
    </div>
    <div>
      <p className="text-xs opacity-80">月額合計</p>
      <p className="text-base font-bold mt-1">{formatYen(monthlyGrandTotal)}</p>
    </div>
    <div>
      <p className="text-xs opacity-80">予約</p>
      <p className="text-base font-bold mt-1">{bookingCount ?? 0}件</p>
    </div>
  </div>
</section>
```

## 実装上の意図

今回のマイページ UI は、単なる一覧ページではなく、
会員さまにとっての「会員活動のホーム画面」として設計しております。
とくに、電話やメールでの問い合わせを減らすため、
「今の状態が見えること」「次に何を押せばよいかが分かること」を重視しました。

そのため、決済異常や停止予定のような重要情報は目立つ位置で表示し、
一方で通常操作は画面内で自然に遷移できる構成にしています。
これにより、サポート工数の削減と、会員さまの自己解決率向上の両立を狙っております。

## ご依頼内容との対応

| いただいたご要望 | 対応状況 |
| --- | --- |
| 会員自身で支援状況を確認できるようにしたい | ログイン直後の 1 画面で、会員種別・支援・決済・予約をまとめて表示 |
| スマホでも見やすくしたい | レスポンシブなグリッドと縦積み UI により、小画面でも読みやすく調整 |
| 支援の変更・停止をすぐ押せる場所に置きたい | 支援中の馬ごとの行に「変更する」「停止する」を直接配置 |
| 決済失敗を見落とさないようにしたい | 状態バナーとお支払い状況欄を上部に配置し、優先的に可視化 |
