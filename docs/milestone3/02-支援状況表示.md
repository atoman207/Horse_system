# 2. 支援状況表示

## 概要

マイページから、会員さまがご自身の「支援中の馬」を
**馬ごとに分けて、口数・月額・ステータスとあわせて一覧表示**できるように実装いたしました。

複数の馬をご支援いただいている会員さまでも、
1 画面ですべての支援状況をご確認いただけます。

また、単に一覧表示するだけでなく、
停止予定の支援、現在有効な支援、月額の合計などを
会員さまが誤解なく理解できるよう、表示ルールを整理しております。
これにより、「今どの馬を支援しているか」「いつまで継続するか」「変更や停止はどこから行うか」が
自然に分かる構成となっております。

## 実装した主な機能

- 支援中の馬を馬ごとに行として表示（馬名・口数・月額・状態）
- 契約（`contracts`）・支援（`support_subscriptions`）・馬（`horses`）を FK で結合して取得
- アクティブな支援のみを抽出し、停止済み支援は「履歴」から参照
- 「停止予定」の支援には琥珀色の `chip-warn` ラベルを付与し、終了予定日を併記
- 管理者画面の顧客詳細と完全に同一のデータソースを参照しているため、
  会員さま／管理者さま両者の見える情報にずれが発生しない
- 同一会員・同一馬の重複支援は別行ではなく統合されるため、一覧が煩雑にならない

## 画面で確認できるポイント

マイページの「支援中の馬」セクションでは、
各馬について「現在何口支援しているか」「月額がいくらか」を
1 行ずつ確認できる構成にしております。

さらに、停止申請済みの支援については、
通常の支援と同じ見た目のままではなく、
「停止予定」の補助ラベルと終了予定日を表示することで、
まだ利用可能な期間が残っていることを誤解なく伝えられるようにしています。

各行の右側には「変更する」「停止する」の導線を用意しており、
一覧を見ながらそのまま次の操作へ進めます。
この構成により、会員さまが事務局へ連絡することなく、
ご自身で内容を確認しながら支援管理を行えるようになっております。

## 実装コード（抜粋）

支援情報はサーバサイドで `loadActiveSupports` により取得し、
停止予定かどうかの判定を行ったうえで UI にレンダリングします。

```ts
// src/lib/customer.ts — 支援の取得
export async function loadActiveSupports(customerId: string) {
  const supabase = createSupabaseServerClient();
  const { data } = await supabase
    .from("support_subscriptions")
    .select(`
      id, status, units, monthly_amount, canceled_at,
      horse:horses ( id, name ),
      contract:contracts ( id, stripe_subscription_id, status )
    `)
    .eq("customer_id", customerId)
    .in("status", ["active", "past_due"])
    .order("created_at", { ascending: true });
  return data ?? [];
}
```

停止予定判定は UI 側で「active かつ canceled_at が未来日付」という条件で行います。

```tsx
// src/app/mypage/page.tsx — 停止予定の抽出
const scheduledStop = supports
  .filter((s) => s.status === "active" && s.canceled_at)
  .map((s) => ({
    id: s.id,
    name: s.horse?.name ?? "—",
    date: s.canceled_at as string,
  }))
  .filter((s) => new Date(s.date).getTime() > now)
  .sort((a, b) => a.date.localeCompare(b.date));
```

支援馬のリスト描画では、馬ごとに「変更する」「停止する」ボタンを並べます。

```tsx
{supports.map((s) => {
  const isScheduledStop =
    s.status === "active" &&
    s.canceled_at &&
    new Date(s.canceled_at).getTime() > now;
  return (
    <li key={s.id} className="py-4 flex flex-col sm:flex-row sm:items-center gap-3">
      <div className="flex-1">
        <p className="font-bold text-lg">
          {s.horse?.name ?? "—"}
          {isScheduledStop && <span className="chip-warn ml-2 text-xs">停止予定</span>}
        </p>
        <p className="text-sm text-ink-soft">
          {formatUnits(s.units)} / {formatYen(s.monthly_amount)} / 月
        </p>
      </div>
      <div className="flex gap-2">
        <Link href={`/mypage/supports/${s.id}`} className="btn-secondary py-2 px-4">
          変更する
        </Link>
        <Link href={`/mypage/supports/${s.id}/stop`} className="btn-ghost py-2 px-4 text-danger">
          停止する
        </Link>
      </div>
    </li>
  );
})}
```

## 実装上の意図

支援状況表示は、会員さまにとって「自分が今どの馬を支えているか」を
確認する最も基本的な機能です。
そのため、実装上は複数テーブルにまたがる情報であっても、
表示上は 1 行 1 支援としてシンプルに整理しております。

また、停止済み・停止予定・有効支援が混在すると、
利用者が混乱しやすくなります。
そこで、現在有効な支援のみを主一覧に出し、
停止予定は補助表示で明示し、
停止済みは履歴側へ切り分けることで、
「今見たい情報」と「過去に見返したい情報」を分離しております。

## ご依頼内容との対応

| いただいたご要望 | 対応状況 |
| --- | --- |
| 会員ごとに支援中の馬がひと目でわかるようにしたい | 馬単位で行を描画し、口数と月額を併記 |
| 停止予定の馬を事前に把握したい | 「停止予定」ラベル＋終了予定日を行内と上部バナーに表示 |
| 管理者が見る情報とずれない状態にしたい | マイページと管理者詳細で同一のクエリ／テーブルを参照 |
| 同じ馬への支援が重複して分かりにくくならないようにしたい | 同一馬の支援は内部的に統合し、一覧表示を簡潔に維持 |
