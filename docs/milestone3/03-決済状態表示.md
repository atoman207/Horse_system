# 3. 決済状態表示

## 概要

会員さまのお支払い状況を、マイページ上で **「正常 / 失敗 / 処理中 / 停止」の 4 分類**に
正規化して常時表示する仕組みを実装いたしました。

Stripe 側で発生する様々なイベント（`succeeded` / `failed` / `past_due` / `canceled` 等）を、
会員さまにとって分かりやすい 4 状態に集約することで、
「自分は今どういう状態か」をひと目で判断いただけます。

## 実装した主な機能

- 決済状態を `ok` / `failed` / `in_progress` / `stopped` の 4 分類に正規化
- 状態ごとに色とラベル（緑／赤／琥珀／灰）を自動割当し、`chip-class` を UI に出力
- 直近決済が「失敗 → 成功」と復帰した場合は「お支払いが復帰しました」の案内バナーを表示
- 決済失敗時は「お支払い情報を変更」ボタン（Stripe Customer Portal）を目立つ位置に表示
- 支援停止予定がある場合は、主状態と矛盾しない補助メッセージを表示

## 実装コード（抜粋）

決済状態の 4 分類正規化は、契約ステータスから導出します。

```ts
// src/lib/paymentStatus.ts
export type PaymentStatusKey = "ok" | "failed" | "in_progress" | "stopped";

export function fromContractStatus(status: string | null | undefined): PaymentStatusKey {
  switch (status) {
    case "active":
      return "ok";
    case "past_due":
      return "failed";
    case "canceled":
      return "stopped";
    case "paused":
    case "incomplete":
    default:
      return "in_progress";
  }
}
```

マイページ側では「契約がなければ支援側の状態で判定」「active で canceled_at 未来なら in_progress」
というロジックを `page.tsx` の冒頭で行います。

```tsx
// src/app/mypage/page.tsx
const baseStatusKey = contract
  ? fromContractStatus(contract.status)
  : supports.length > 0
    ? fromContractStatus(supports[0].status)
    : "stopped";

// 直近決済成功からの「復帰」案内（失敗→成功の救済表示）
const recentlyRecovered =
  baseStatusKey === "ok" &&
  recentPayments.some((p) => p.status === "failed") &&
  recentPayments[0]?.status === "succeeded";

const display = describePaymentDisplay(displayKey, {
  recentlyRecovered,
  scheduledCancelAt: scheduledStop[0]?.date ?? null,
});
```

画面上のバナー表示も、状態に応じて自動で切り替わります。

```tsx
{displayKey === "failed" && (
  <div className="mt-3 p-3 rounded-xl bg-red-50 border-2 border-red-200">
    <p className="font-bold text-danger">お支払いが完了していません</p>
    <p className="text-sm mt-1">
      「お支払い情報を変更」ボタンからカード情報をご確認ください。
    </p>
  </div>
)}
```

Stripe 側での状態変化については、Webhook 受信時に契約・支援の状態を
DB へ反映することで、マイページ表示と実際の課金状態のずれを防いでいます。

```ts
if (event.type === "invoice.payment_succeeded" && contract) {
  const nextPeriodEnd = invoice.lines?.data?.[0]?.period?.end;
  await admin
    .from("contracts")
    .update({
      status: "active",
      current_period_end: nextPeriodEnd
        ? new Date(nextPeriodEnd * 1000).toISOString()
        : undefined,
    })
    .eq("id", contract.id);
}
```

## 画面で確認できるポイント

会員さまはマイページにログインした時点で、
ご自身のお支払い状況を日本語の 4 分類で確認できます。
これにより、Stripe 独自の `past_due` や `incomplete` といった表現を理解しなくても、
「今は正常なのか」「確認が必要なのか」が直ちに分かる状態となっております。

また、決済失敗の際には説明文と再設定導線を併記しているため、
単に「失敗」と表示するだけで終わらず、
会員さまが次に取るべき行動まで案内できる構成になっております。
さらに、失敗後に復帰した場合もその旨を補助表示することで、
利用者の不安を軽減する設計としています。

## 実装上の意図

決済状態表示は、単なるデータ表示ではなく、
会員さまへの「安心」と「行動導線」を両立させる機能として設計しております。
たとえば、失敗状態を見つけた際にすぐカード更新へ進めること、
復帰後には正常化したことが分かること、
停止予定は即停止と誤認されないことが重要でした。

そのため、Stripe の生イベントをそのまま見せるのではなく、
アプリ側で UI 表示用の状態へ正規化し、
文言・色・補助メッセージまで含めて統一しております。
