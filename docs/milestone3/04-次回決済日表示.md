# 4. 次回決済日表示

## 概要

会員さまのマイページに、
**次回の自動決済が行われる日付**を Stripe の情報をもとに常時表示する機能を実装いたしました。

Stripe が管理している `current_period_end`（現在の課金期間の終了日）を最優先で参照することで、
期中での口数変更や日割り精算が発生した場合でも、
常に「Stripe 側と一致した正確な次回決済日」が表示されます。

## 実装した主な機能

- 契約（`contracts.current_period_end`）を最優先で参照
- 契約が未設定の場合は顧客サマリ（`customer_summary.next_payment_at`）を fallback
- 停止済み・未契約の会員には「現在、継続課金中のご契約はありません」と案内
- 失敗中・処理中でも日付は継続表示し、状態バナーと併用で意味を補足
- 表示形式は会員が読み取りやすい日付表記へ統一

## 実装コード（抜粋）

マイページ側での「次回決済日」導出ロジックです。

```tsx
// src/app/mypage/page.tsx
// --- 次回決済日 ---
//   Stripe の current_period_end を最優先。
//   継続課金がない場合は空欄扱いとして
//   「現在、継続課金中のご契約はありません」を表示。
const nextPaymentAt =
  contract?.current_period_end ?? summary?.next_payment_at ?? null;

const hasActiveRecurring =
  Boolean(nextPaymentAt) &&
  (displayKey === "ok" || displayKey === "failed" || displayKey === "in_progress");
```

UI 描画部分では、契約がない場合は日付を出さずに案内文に切り替えます。

```tsx
<div>
  <p className="label">次回決済日</p>
  {hasActiveRecurring ? (
    <p className="text-lg">{formatDate(nextPaymentAt, false)}</p>
  ) : (
    <p className="text-sm text-ink-soft">現在、継続課金中のご契約はありません</p>
  )}
</div>
```

`current_period_end` は、Stripe Webhook `customer.subscription.updated` を
受信したタイミングで DB 側に同期されます。

```ts
const currentPeriodEndIso = sub.current_period_end
  ? new Date(sub.current_period_end * 1000).toISOString()
  : null;

await admin.from("contracts").upsert(
  {
    customer_id: customer.id,
    stripe_subscription_id: sub.id,
    status: mappedStatus,
    current_period_end: currentPeriodEndIso,
  },
  { onConflict: "stripe_subscription_id" },
);
```

## 画面で確認できるポイント

会員さまはマイページ上で、
「次回いつ引き落としが行われるのか」を
他の情報に埋もれずに確認できます。
また、継続課金契約がない場合には日付欄を空のままにするのではなく、
「現在、継続課金中のご契約はありません」という案内文へ切り替えることで、
誤解のない表示となるようにしております。

さらに、決済失敗中や手続き中であっても、
契約が継続している限りは次回決済予定日を併記するため、
会員さまにとって「次にいつ課金が走る可能性があるか」が把握しやすい構成となっています。

## 実装上の意図

次回決済日表示は、単に日付を見せる機能ではなく、
会員さまの不安を減らすための重要な情報設計と考えております。
月額支援を行う会員さまにとって、
「次はいつ決済されるのか」が明確であることは、
継続利用への安心感に直結します。

一方で、契約が存在しない会員さまや停止済み会員さまに
誤って日付を表示してしまうと、大きな混乱につながります。
そのため、契約の有無・状態・決済状態と組み合わせて表示条件を制御し、
必要な場合にだけ、正しい意味をもって日付を出すようにしています。

## ご依頼内容との対応

| いただいたご要望 | 対応状況 |
| --- | --- |
| 次の引き落とし日が会員にわかるようにしたい | マイページ常設表示（`2026年5月1日` 形式） |
| 口数変更後のズレが起きないようにしたい | Stripe 側の `current_period_end` を最優先で反映 |
| 継続課金していない会員に誤表示したくない | 契約がない場合は日付を出さず案内文に差し替え |
| 状態表示と矛盾しないようにしたい | 決済状態と組み合わせて表示可否を判定し、意味のある日付のみ表示 |
