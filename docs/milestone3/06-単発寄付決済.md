# 6. 単発寄付決済

## 概要

継続支援とは別に、**一回限りの寄付**を会員さま／非会員さまの双方から
受け付けられる仕組みを実装いたしました。

寄付金額は任意入力のほか、プリセット（1,000 円／3,000 円／5,000 円／10,000 円）からも選択可能で、
決済は Stripe Checkout に委譲するため、
クレジットカード情報がサーバ側に直接流入しない PCI-DSS 準拠の構成です。

## 実装した主な機能

- 会員／非会員の両方から寄付可能（会員向け導線と一般向け導線を分離）
- 金額はプリセットから選択／任意入力の両対応
- 一般寄付ではメールアドレスを必須取得し、確認メールと将来の履歴統合に対応
- 決済は Stripe Checkout（`mode: payment`）へリダイレクト
- Webhook `checkout.session.completed` 受信時に `donations.status` を `succeeded` へ更新
- `payments` テーブルに `kind: donation` で明細を追加し、顧客詳細から横断検索可能
- 成功時は `/mypage/donate/thanks` へ遷移、お礼メールを自動配信（→ `07-自動お礼メール.md`）
- 一般寄付者のメールアドレスから既存会員データとの紐付け候補を判定可能

## 実装コード（抜粋）

### 寄付申込 API（Stripe Checkout セッション生成）

```ts
// src/app/api/donations/route.ts
export async function POST(req: NextRequest) {
  const body = await req.json();
  const parsed = schema.safeParse(body);
  if (!parsed.success) {
    return NextResponse.json({ error: parsed.error.issues[0]?.message }, { status: 400 });
  }

  const { amount, donor_name, donor_email, message } = parsed.data;
  const admin = createSupabaseAdminClient();
  const session = await getSession();

  const { data: donation } = await admin
    .from("donations")
    .insert({
      customer_id: session?.customerId ?? null,
      donor_name: donor_name ?? null,
      donor_email: donor_email ?? null,
      amount,
      message: message ?? null,
      status: "pending",
    })
    .select()
    .single();

  const stripe = getStripe();
  const checkout = await stripe.checkout.sessions.create({
    mode: "payment",
    payment_method_types: ["card"],
    line_items: [{
      price_data: {
        currency: "jpy",
        product_data: { name: "Retouchメンバーズサイト 単発寄付" },
        unit_amount: amount,
      },
      quantity: 1,
    }],
    client_reference_id: donation.id,
    metadata: { donation_id: donation.id },
    customer_email: donor_email ?? undefined,
    success_url: `${process.env.NEXT_PUBLIC_SITE_URL}${returnBase}/thanks?sid={CHECKOUT_SESSION_ID}`,
    cancel_url: `${process.env.NEXT_PUBLIC_SITE_URL}${returnBase}`,
  });

  return NextResponse.json({ url: checkout.url });
}
```

### Webhook での寄付確定処理

```ts
// src/app/api/stripe/webhook/route.ts（抜粋）
case "checkout.session.completed": {
  const session = event.data.object as Stripe.Checkout.Session;
  const donationId = session.metadata?.donation_id ?? session.client_reference_id;
  if (session.mode === "payment" && donationId) {
    await admin
      .from("donations")
      .update({
        status: "succeeded",
        stripe_payment_intent_id: session.payment_intent as string,
        donated_at: new Date(session.created * 1000).toISOString(),
      })
      .eq("id", donationId);

    // payments にも明細行を追加（履歴紐付け基盤と連携）
    const { data: donation } = await admin
      .from("donations").select("*").eq("id", donationId).maybeSingle();
    await admin.from("payments").insert({
      customer_id: donation.customer_id,
      donation_id: donation.id,
      kind: "donation",
      amount: donation.amount,
      currency: "jpy",
      status: "succeeded",
      stripe_event_id: event.id,
      stripe_payment_intent_id: session.payment_intent as string,
      occurred_at: new Date(session.created * 1000).toISOString(),
    });

    // お礼メール自動配信（07-自動お礼メール.md 参照）
    const tpl = donationThanksTemplate({
      name: donation.donor_name,
      amount: donation.amount,
    });
    await notify({
      kind: "donation_thanks",
      to: donation.donor_email,
      to_name: donation.donor_name,
      ...tpl,
    });
  }
  break;
}
```

## 画面で確認できるポイント

会員さま向けの単発寄付画面では、
マイページにログインした状態のまま寄付金額を選択し、
必要に応じて応援メッセージを添えて寄付へ進むことができます。
継続支援とは別の一回限りの応援であることが明確に書かれているため、
月額支援との混同が起きにくい構成です。

一般寄付者さま向け画面では、
お名前、メールアドレス、応援メッセージとあわせて寄付申込ができ、
メールアドレスを必須とすることで、
完了通知の送付と、将来的な会員化後の履歴統合に備えております。

決済完了後は完了画面へ遷移し、
受付完了と確認メール送信の案内を表示するため、
寄付者さまにとって「申込が完了したかどうか」が分かりやすい流れになっています。

## 実装上の意図

単発寄付は、継続支援よりも気軽に利用される機能である一方、
決済の安全性と履歴の一貫性が非常に重要です。
そのため、カード情報はアプリケーション側で保持せず、
Stripe Checkout に全面委譲する方式を採用しております。

また、寄付申込時点でまず `donations` テーブルに `pending` レコードを保存し、
Webhook 成功時に `succeeded` へ更新することで、
途中離脱・キャンセル・成功後反映の各状態を追跡できるようにしています。
これにより、問い合わせ対応や運営確認の際にも、寄付の状態を明確に説明できます。

## ご依頼内容との対応

| いただいたご要望 | 対応状況 |
| --- | --- |
| 継続支援とは別に一回限りの寄付を受けたい | 会員向け・一般向けの両方で単発寄付導線を実装 |
| クレジット情報をサーバで保持したくない | Stripe Checkout に委譲（PCI-DSS 準拠） |
| 寄付履歴を会員ごとに確認したい | `payments.kind='donation'` で履歴紐付け基盤と自動連携 |
| お礼メールを自動送信したい | Webhook 成功時に `donationThanksTemplate` で自動送信 |
| 一般寄付者も後から履歴統合できるようにしたい | メールアドレス必須取得により、将来の会員紐付けに対応 |
