# 1. 定員管理と空き表示

## 概要

Milestone 4 では、見学会・個別見学の運用に必要な「定員管理」と「空き表示」を実装いたしました。

各イベントに定員を設定し、予約済み人数をもとに残席を自動計算することで、会員さまは申し込み前に空き状況を確認できます。管理者さまは、管理画面からイベントごとの定員、予約席数、残席を確認できるため、予約受付状況を即座に把握できます。

## 実装した内容

| 項目 | 状態 | 内容 |
| --- | --- | --- |
| イベント別定員 | 完了 | `events.capacity` により、見学会・個別見学ごとに定員を保持 |
| 予約席数の集計 | 完了 | `bookings.party_size` を合計して予約済み席数を算出 |
| キャンセル除外 | 完了 | `status = canceled` の予約は予約席数から除外 |
| 残席表示 | 完了 | 定員から予約席数を差し引いて残席を表示 |
| 満席制御 | 完了 | 残席が 0 のイベントは会員画面で「満席」と表示 |
| 変更時の再計算 | 完了 | 人数変更時、自分自身の現在予約分を除外して空き確認 |

## 会員画面での表示

会員さまは `/mypage/bookings` にて、公開中の見学会・個別見学を確認できます。

各イベントには、以下の情報を表示しています。

- イベント名
- 見学会／個別見学の種別
- 開催日時
- 開催場所
- 定員
- 残席
- 支援者限定イベントかどうか
- 満席／対象外／予約中の状態

残席がある場合は予約ボタンを表示し、残席がない場合は「満席」と表示します。これにより、会員さまが申し込み可能なイベントを直感的に判断できます。

## 管理画面での表示

管理者さまは `/admin/events/[id]` と `/admin/bookings` から、イベントごとの定員・予約席数・残席を確認できます。

イベント詳細画面では、以下をカード形式で表示しています。

- 定員
- 予約席数
- 残席

予約管理画面では、選択したイベントに対して、定員、予約数、残席を一覧上部に表示しています。これにより、電話や個別連絡で予約を受けた場合でも、管理者さまが残席を確認しながら代理登録できます。

## 技術的な実装

定員と残席の計算は `src/lib/bookings.ts` に共通化しています。

主な関数は以下です。

| 関数 | 用途 |
| --- | --- |
| `seatUsage()` | 単一イベントの定員・予約席数・残席を計算 |
| `seatUsageBatch()` | 複数イベントの予約席数をまとめて集計 |

`seatUsage()` では、対象イベントの予約レコードを読み込み、キャンセル済み予約を除外したうえで `party_size` を合計します。人数変更時には、現在の会員自身の予約分を一度除外し、変更後の人数で再計算します。これにより、既存予約の変更時に自分の席を二重に数えることを防いでいます。

## 完了確認

以下の動作を確認済みです。

- 定員が設定されたイベントを作成できること
- 予約人数に応じて残席が減ること
- キャンセル済み予約が残席計算から除外されること
- 残席が 0 の場合、会員画面で満席表示になること
- 定員を超える予約が API 側で拒否されること
- 管理画面で定員・予約席数・残席を確認できること

以上により、Milestone 4 における定員管理と空き表示は完了しております。
