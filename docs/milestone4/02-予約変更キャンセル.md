# 2. 予約・変更・キャンセル

## 概要

Milestone 4 では、会員さまがマイページから見学会・個別見学の予約、予約内容の変更、キャンセルを行える機能を実装いたしました。

あわせて、管理者さまが管理画面から代理予約、予約内容の変更、キャンセル、削除を行えるようにし、オンライン操作と事務局対応の両方に対応できる運用基盤を整備しています。

## 実装した内容

| 項目 | 状態 | 内容 |
| --- | --- | --- |
| 会員による予約 | 完了 | 公開中イベントを選択し、人数・メモを入力して予約可能 |
| 会員による変更 | 完了 | 自分の予約について、人数・メモを変更可能 |
| 会員によるキャンセル | 完了 | 自分の予約をキャンセル可能 |
| 管理者による代理予約 | 完了 | 管理画面から会員を検索し、イベントに予約を追加可能 |
| 管理者による変更 | 完了 | 人数、メモ、状態を管理画面から変更可能 |
| 管理者による取消 | 完了 | 履歴を残したまま予約をキャンセル可能 |
| 管理者による削除 | 完了 | 必要な場合のみ予約レコードの完全削除に対応 |
| 重複予約防止 | 完了 | 同一会員・同一イベントの重複予約を防止 |

## 会員向け予約フロー

会員さまは `/mypage/bookings` から予約できます。

画面には公開中のイベント一覧が表示され、予約可能なイベントには「予約する」ボタンを表示します。ボタンを押すと、人数とメモを入力するフォームが開きます。

入力後、「予約を確定」を押すことで予約が作成されます。予約作成時には、以下を API 側で確認しています。

- ログイン済み会員であること
- 対象イベントが存在し、公開中であること
- 支援者限定イベントの場合、会員が支援中であること
- 定員を超えないこと
- 同じイベントをすでに予約していないこと

予約が完了すると、予約内容が「自分の予約」欄に表示されます。

## 会員向け変更フロー

会員さまは、自分の予約一覧から「内容変更」を押すことで、予約人数とメモを変更できます。

人数変更時には、定員チェックを再実行します。このとき、自分自身の現在予約分は一度除外し、変更後の人数を加えて定員を超えないか確認します。これにより、既存予約の人数変更時にも正しく残席を判定できます。

## 会員向けキャンセルフロー

会員さまは、自分の予約一覧から「キャンセル」を押すことで予約を取り消せます。

キャンセルはレコードを削除するのではなく、`status = canceled` と `canceled_at` を保存するソフトキャンセル方式です。これにより、後からキャンセル履歴を確認できます。

## 管理者向け予約管理

管理者さまは `/admin/bookings` からイベント別に予約を管理できます。

主な操作は以下です。

- イベントを選択して予約一覧を表示
- 会員検索による代理予約
- 人数変更
- メモ変更
- 状態変更
- 予約取消
- 必要時の完全削除

管理者による予約追加では、通常は定員・支援者限定チェックを行います。ただし、運用上の特別対応が必要な場合に備え、管理者権限でチェックを上書きする `bypass_guard` にも対応しています。

## 技術的な実装

会員向け予約 API は `src/app/api/mypage/bookings/route.ts` に実装しています。

| メソッド | 内容 |
| --- | --- |
| `POST` | 予約作成 |
| `PATCH` | 予約人数・メモの変更 |
| `DELETE` | 予約キャンセル |

管理者向け予約 API は以下に実装しています。

| ファイル | 内容 |
| --- | --- |
| `src/app/api/admin/bookings/route.ts` | 管理者による予約作成 |
| `src/app/api/admin/bookings/[id]/route.ts` | 管理者による変更・取消・削除 |

入力検証には `zod` を使用し、API 境界で不正な入力を拒否しています。

## 完了確認

以下の動作を確認済みです。

- 会員が公開中イベントを予約できること
- 同一会員が同じイベントを重複予約できないこと
- 会員が自分の予約人数とメモを変更できること
- 会員が自分の予約をキャンセルできること
- 管理者が代理予約を作成できること
- 管理者が予約内容を変更できること
- 管理者が予約をキャンセルできること
- 定員超過時に予約・変更が拒否されること

以上により、Milestone 4 における予約・変更・キャンセル機能は完了しております。
