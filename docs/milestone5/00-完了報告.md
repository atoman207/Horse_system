# マイルストーン 5：テスト・CSV移行・公開対応 完了報告

実装日: 2026-04-23

## 1. 既存 CSV 移行
- 既存スプレッドシート → Supabase の取込は `scripts/seed-from-xlsx.ts`（既存）。冪等で繰り返し実行可。
- 管理画面 `/admin/csv` から CSV 入出力：顧客・支援・寄付（既存）に加え以下を追加。
  - イベント CSV: `GET /api/admin/csv/events`（定員・予約席数・残席を含む）
  - 予約 CSV:   `GET /api/admin/csv/bookings`（`?event_id=` で絞り込み可）
- すべての CSV は UTF-8 + BOM で出力するため Excel でそのまま開ける。

## 2. 結合テスト
- テストランナーは Node 20+ 標準の `node:test` を `tsx --test` で実行（追加依存ゼロ）。
- 26 ケース（うち主要なもの）:
  | カテゴリ | 件数 | 内容 |
  |---|---|---|
  | CSV | 6 | エスケープ、ヘッダトリム、空入力、ラウンドトリップ |
  | フォーマッタ | 5 | 円・日付・口数・性別・状態 |
  | 予約ヘルパー | 8 | `seatUsage`（除外含む）／`seatUsageBatch`／`hasActiveSupport` |
  | 予約シナリオ | 6 | 定員、支援者限定、キャンセル復帰、自分の席除外 |
  | 監査ログ | 1 | DB が落ちていても呼び出し元を壊さない |
- 実行: `npm test`
- live DB に対するエンドツーエンド: `npm run smoke:bookings`（テストイベント・予約を作って自動クリーンアップ）。

## 3. 公開準備
- `npm run preflight` を新設。以下を一括チェックし、欠けがあれば exit 1。
  1. 必須環境変数（`NEXT_PUBLIC_SUPABASE_URL`、`SUPABASE_SERVICE_ROLE_KEY`、Stripe 三点、`NEXT_PUBLIC_SITE_URL`）
  2. Supabase に接続できること
  3. すべてのテーブル + `v_customer_summary` ビューが存在
  4. `membership_plans` と `horses` のシードが入っていること
  5. `audit_logs` が書き込み可能（プローブ行を入れて即削除）
- `npm run smoke` は引き続きスキーマの存在確認のみを行う軽量チェック。
- デプロイ手順は本フォルダ `01-デプロイ手順.md` を参照。

## 4. 初期調整
- 監査ログのアクション一覧を追加（`event.*`、`booking.*`、`booking.self_*`）。
- 監査ログのテーブルフィルタに `events` を追加。
- 顧客詳細・イベント詳細ページから「予約変更／キャンセル履歴」を即座に確認可能に。
- 管理画面「予約追加」フォームを UUID 直入力 → 顧客名／メール検索に置き換え。誤登録を低減。
- 会員予約 UI で残席を考慮した最大人数を表示し、超過入力を防止。
