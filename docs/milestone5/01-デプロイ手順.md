# デプロイ手順 / リリースチェックリスト

このプロジェクトは Next.js 14 (App Router) + Supabase + Stripe の構成です。
本番リリース時は以下の手順を順に実行してください。

## 1. リポジトリの準備
- [ ] `main` ブランチが最新であること
- [ ] `npm install` が成功すること
- [ ] `npm run lint` がエラーなく通ること
- [ ] `npm test` が全件 PASS であること（26 ケース）

## 2. 環境変数の設定
本番環境（Vercel / Render / 自社サーバなど）に以下を設定。

| キー | 必須 | 用途 |
|---|:-:|---|
| `NEXT_PUBLIC_SUPABASE_URL` | ✅ | Supabase URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | ✅ | Supabase anon key（クライアント） |
| `SUPABASE_SERVICE_ROLE_KEY` | ✅ | Supabase service role key（サーバ専用） |
| `STRIPE_SECRET_KEY` | ✅ | Stripe シークレットキー |
| `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` | ✅ | Stripe 公開可能キー |
| `STRIPE_WEBHOOK_SECRET` | ✅ | Stripe Webhook 署名シークレット |
| `NEXT_PUBLIC_SITE_URL` | ✅ | サイト URL（メール／Stripe URL に使用） |
| `NOTIFY_TRANSPORT` |   | `smtp` / `resend` / `audit`（既定 `smtp`） |
| `SMTP_HOST` `SMTP_USER` `SMTP_PASS` `SMTP_PORT` `SMTP_SECURE` |   | SMTP 利用時 |
| `MAIL_FROM` `MAIL_FROM_NAME` |   | 送信元 |
| `RESEND_API_KEY` |   | Resend 利用時 |

## 3. Supabase スキーマ適用
- [ ] Supabase SQL Editor で `supabase/apply_all.sql` の内容を貼り付け実行
- [ ] テーブル＋ビュー＋RLS ポリシーが有効になっていることを Supabase ダッシュボードで確認

## 4. 既存 CSV データ移行
- [ ] `npm run seed:xlsx` を本番接続で実行（顧客／支援サブスク／契約／会員プランをインポート）
- [ ] 必要に応じて `/admin/csv` から差分 CSV を取込

## 5. Stripe Webhook 設定
- 受信エンドポイント: `https://<本番ホスト>/api/stripe/webhook`
- 受信イベント:
  - `checkout.session.completed`
  - `invoice.payment_succeeded`
  - `invoice.payment_failed`
  - `customer.subscription.created` / `updated` / `deleted`
  - `payment_intent.succeeded`
  - `payment_intent.payment_failed`
- [ ] テストイベントを送って `audit_logs` に記録されることを確認

## 6. プリフライトチェック
- [ ] `npm run preflight` を本番接続で実行し、すべて ✅ になること
- [ ] `npm run smoke:bookings` で予約フローのエンドツーエンド確認

## 7. 初期管理者の作成
1. Supabase Auth でメンテナンス担当者のユーザを作成
2. `profiles` テーブルで `role = 'admin'` に更新
3. `/admin/login` からログインできることを確認

## 8. 本番デプロイ
- [ ] `npm run build` がローカルで成功すること
- [ ] CI / ホスティング側で同じビルドが成功すること
- [ ] `/admin` にログインしてダッシュボードが表示されること
- [ ] `/mypage/bookings` で予約が動作すること

## 9. リリース後モニタリング（最初の 24 時間）
- [ ] `/admin/audit-logs` で `notify.*`、`booking.*`、`payment.*` が記録されていること
- [ ] Stripe ダッシュボードでイベント受信に失敗が無いこと
- [ ] メール（SMTP / Resend）が遅延なく届くこと

## ロールバック手順
1. ホスティング側で直前のデプロイにロールバック
2. Stripe Webhook は同じシークレットを共有するため変更不要
3. Supabase はマイグレーション破壊的変更を入れない限り後方互換
