## 既存 CSV 移行

既存スプレッドシート（`docs/` 配下の `.xlsx`）からの一括取込は、`npm run seed:xlsx`（`scripts/seed-from-xlsx.ts`）で実行できます。本スクリプトは冪等であり、同じデータに対して何度実行しても重複が発生しません。

- 顧客はメールアドレスをキーにアップサート
- 馬は馬名をキーにアップサート
- 契約・支援サブスクリプションは既存レコードがある場合は再作成しない
- Excel の日付シリアル値、半口（0.5口）、ランク（メンバーズ／アテンダー／オーナーズ／支援／半口など）から会員プランへのマッピングに対応

公開後の差分取込・修正対応は、管理画面 `/admin/csv` から個別の CSV ファイルで実施できます。顧客 CSV・支援 CSV はインポートに対応し、寄付 CSV・イベント CSV・予約 CSV はエクスポート専用としております。すべての CSV は UTF-8 + BOM 付きで出力されるため、Excel でそのまま開くことができます。

## 結合テスト

自動テストは、Node 20+ に標準搭載されている `node:test` と `tsx --test` を使用し、追加依存ゼロで実行できる構成としております。`npm test` で全 26 ケースを一括実行できます。

| カテゴリ | 件数 | 内容 |
| --- | --- | --- |
| CSV | 6 | エスケープ、ヘッダトリム、空入力、ラウンドトリップ |
| フォーマッタ | 5 | 円・日付・口数・性別・状態 |
| 予約ヘルパー | 8 | 座席使用数（除外含む）、バッチ取得、支援有無判定 |
| 予約シナリオ | 6 | 定員到達、支援者限定、キャンセル復帰、自分の席除外 |
| 監査ログ | 1 | DB 障害時でも呼び出し元を壊さないことの保証 |

加えて、`npm run smoke:bookings` では実 Supabase 接続に対してテストイベント・テスト予約を作成し、自動でクリーンアップする End-to-End スモークテストも整備しております。

## 公開準備

`npm run preflight` を新設し、公開直前に以下を一括チェックする仕組みを導入しました。欠けがあれば終了コード 1 で停止しますので、CI／デプロイ前ゲートとして利用いただけます。

1. 必須環境変数（`NEXT_PUBLIC_SUPABASE_URL`、`SUPABASE_SERVICE_ROLE_KEY`、Stripe 三点、`NEXT_PUBLIC_SITE_URL`）
2. Supabase への接続可否
3. すべてのテーブル＋`v_customer_summary` ビューの存在
4. `membership_plans` と `horses` の初期シード投入状況
5. `audit_logs` への書き込み可否（プローブ行を入れて即削除）

`npm run smoke` は引き続きスキーマの存在確認のみを行う軽量チェックとして残しております。本番リリース時のデプロイ手順・チェックリストは `docs/milestone5/01-デプロイ手順.md` にまとめております。

## 初期調整

運用開始後に発生しうる操作性・確認性の課題について、以下の初期調整を実施しております。

- 監査ログのアクション一覧を拡張（`event.*`、`booking.*`、`booking.self_*`）し、見学会・予約周りの操作を網羅
- 監査ログのテーブルフィルタに `events` を追加し、イベント設定変更の追跡を容易化
- 顧客詳細・イベント詳細ページから「予約変更／キャンセル履歴」を直接参照可能に
- 管理画面「予約追加」フォームを UUID 直入力から、顧客名・メールでのサジェスト検索に変更し、誤登録を低減
- 会員予約 UI で残席を考慮した最大人数を表示し、定員超過の入力を防止

## 本番公開に向けての準備状況

上記実装の完了により、**本番公開・本格運用を開始できる状態が整いました。**

本番リリースに際しては、`docs/milestone5/01-デプロイ手順.md` に沿って、以下の流れで進行いただけます。

- リポジトリの準備（`npm install`、`npm run lint`、`npm test` の確認）
- 本番環境変数の設定（Supabase、Stripe、サイト URL、メール送信設定）
- Supabase スキーマの適用（`supabase/apply_all.sql`）
- 既存 CSV データの本番取込（`npm run seed:xlsx`、必要に応じて `/admin/csv` から差分取込）
- Stripe Webhook 設定（本番エンドポイント・受信イベントの登録）
- `npm run preflight` による本番接続の最終チェック
- 初期管理者ユーザの作成
- 本番デプロイおよびリリース後 24 時間のモニタリング

Stripe につきましては、現在テストアカウントで構築済みのため、貴社の本番 Stripe アカウントへのキー切替時に、Supabase 側に保存されているテストモードの ID（`stripe_customer_id`、`stripe_subscription_id`、`stripe_price_id` 等）を初期化する必要がございます。当該切替作業は、 弊社にて代行可能です。
